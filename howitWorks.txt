Есть 2 версии сборки  - dev для разработки и docs для продакшна. В папке gulp находятся наши галп таски, дев для разработки и докс для продакшна. 


 !! ATTENTION !! 
- НЕЛЬЗЯ имена картинок чтоб были через пробел!!!
- Не забывать давать имена кнопкам типа <button id="combo" aria-label="Aria Name">Name</button>
- Везде должен быть href="##", если не нужно никаких действий.
- node modules НИКОГДА НЕ ГРУЗИМ В ГИТ
 !! ATTENTION !! 



 1. Дев версия сбирается в папку build. Запускается командой gulp. НЕ минифицирует картинки для дев.  // .pipe(imagemin({ verbose: true })).это закомментировано, можно включить в дев.жс. То же самое // .pipe(babel()) закометен для дев. Его можно включить, если мы хотим тестировать дев в старых браузерах.


 2. Продакшн версия собирается в папку docs. Называется именно так, потому что git pages запускается по умолчанию из папки докс. Запускается командой gulp docs.

!! КАК ОТДАВАТЬ ПРОЕКТ ЗАКАЗЧИКУ
 1. ели просто загружаем cайт на хостинг, то грузим папку докс. все оптимизировано, все работает;
